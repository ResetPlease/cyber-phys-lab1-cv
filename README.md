# Отчет по лабораторной работе 1

## Тема

Классификация изображений фисташек на наборе данных `Pistachio Image Dataset`.

В работе решалась задача бинарной классификации изображений двух сортов фисташек:

- `Kirmizi_Pistachio`
- `Siirt_Pistachio`

Исходный notebook: `lab1_pistachio.ipynb`.

## Цель работы

Цель лабораторной работы — построить полный pipeline компьютерного зрения для задачи классификации изображений: загрузить датасет, изучить его структуру, подготовить данные, обучить несколько моделей, сравнить качество и сделать выводы по результатам экспериментов.

## Используемые библиотеки

В начале notebook добавлена установка основных библиотек:

- `kagglehub` для загрузки датасета;
- `torch`, `torchvision` для обучения моделей;
- `pandas`, `numpy` для подготовки таблиц и метрик;
- `matplotlib`, `seaborn` для визуализации;
- `scikit-learn` для разбиения данных и расчета метрик;
- `pillow` для чтения изображений.

Также зафиксирована версия `urllib3<2`, чтобы убрать предупреждение `NotOpenSSLWarning` на macOS с LibreSSL.

## Конфигурация эксперимента

В notebook создана единая конфигурация `Config`, которая задает параметры всех экспериментов:

| Параметр | Значение |
|---|---:|
| `seed` | 42 |
| `image_size` | 224 |
| `batch_size` | 32 |
| `baseline_epochs` | 5 |
| `improved_epochs` | 6 |
| `custom_epochs` | 8 |
| `learning_rate` | 0.0001 |
| `weight_decay` | 0.0001 |
| `train/val/test` | 70% / 15% / 15% |

Для воспроизводимости фиксируются random seed для Python, NumPy и PyTorch. В текущем запуске использовалось устройство `mps`, то есть Apple Metal backend.

Датасет загружается через `kagglehub`:

```text

```

После загрузки изображения находятся во вложенной директории:


В notebook используется заранее известная структура датасета, поэтому классы берутся напрямую из двух папок:

| Класс | Количество изображений |
|---|---:|

Дополнительные `.arff`, `.xls`, `.xlsx` файлы в архиве относятся к табличным признакам и в pipeline классификации изображений не используются.

## Подготовка данных

В notebook выполнены следующие шаги подготовки:

1. Собран список путей к изображениям `*.jpg` по двум папкам классов.
2. Создан `DataFrame` с колонками `path` и `label`.
3. Построено распределение классов.
4. Выведены случайные примеры изображений из каждого класса.
5. Данные разбиты стратифицированно на train, validation и test.

Размеры выборок:

| Выборка | Размер |
|---|---:|
| Train | 1503 |
| Validation | 322 |
| Test | 323 |

Стратификация сохраняет пропорции классов во всех частях данных.

## Dataset и DataLoader

Для PyTorch реализован класс `PistachioDataset`, который:

- получает `DataFrame` с путями и метками;
- открывает изображение через `PIL.Image`;
- переводит текстовую метку в числовой индекс;
- применяет transforms;
- возвращает тензор изображения, метку и путь к файлу.

Отображение классов:

| Индекс | Класс |
|---:|---|
| 0 | `Kirmizi_Pistachio` |
| 1 | `Siirt_Pistachio` |

Для notebook на macOS установлен `num_workers=0`, потому что multiprocessing может приводить к ошибке сериализации класса `PistachioDataset` из `__main__`.

## Метрики и обучение

Для оценки используются:

- accuracy;
- precision;
- recall;
- F1-score;
- confusion matrix;
- classification report.

Training loop сохраняет checkpoint лучшей модели по validation F1. Это важно, потому что F1 лучше отражает качество при дисбалансе классов, чем одна accuracy.

## Эксперименты

В notebook обучены и сравнены несколько моделей.

### ResNet-18 baseline

Используется предобученная `ResNet-18` из `torchvision`. Последний классификационный слой заменен на слой для двух классов.

Результат на test:

| Метрика | Значение |
|---|---:|
| test loss | 0.0637 |
| accuracy | 0.9721 |
| precision | 0.9708 |
| recall | 0.9638 |
| F1 | 0.9673 |

### Transformer-like baseline

В качестве transformer-like архитектуры используется `ViT-B-16`. Модель также адаптирована под два класса.

Результат на test:

| Метрика | Значение |
|---|---:|
| test loss | 0.0608 |
| accuracy | 0.9845 |
| precision | 0.9854 |
| recall | 0.9783 |
| F1 | 0.9818 |

### ResNet-18 со сильными аугментациями

К базовым transforms добавлены:

- случайные повороты;
- `ColorJitter`;
- перспективные искажения;
- горизонтальное отражение.

Результат на test:

| Метрика | Значение |
|---|---:|
| test loss | 0.0722 |
| accuracy | 0.9814 |
| precision | 1.0000 |
| recall | 0.9565 |
| F1 | 0.9778 |

### ResNet-18 с class weights и scheduler

В этом эксперименте использованы:

- веса классов для `CrossEntropyLoss`;
- `CosineAnnealingLR`;
- `AdamW`.

Рассчитанные веса классов:

```text
[0.8718097, 1.1723869]
```

Результат на test:

| Метрика | Значение |
|---|---:|
| test loss | 0.0478 |
| accuracy | 0.9814 |
| precision | 0.9925 |
| recall | 0.9638 |
| F1 | 0.9779 |

### Собственная CNN baseline

Дополнительно реализована собственная CNN-модель без использования готовой архитектуры. Она содержит несколько сверточных блоков с `Conv2d`, `BatchNorm2d`, `ReLU`, pooling и полносвязную классификационную голову.

Результат на test:

| Метрика | Значение |
|---|---:|
| test loss | 0.2055 |
| accuracy | 0.9195 |
| precision | 0.8636 |
| recall | 0.9638 |
| F1 | 0.9110 |

### Улучшенная собственная CNN

Для собственной CNN дополнительно применены сильные аугментации, weighted loss и scheduler. В этом запуске такая комбинация ухудшила качество.

Результат на test:

| Метрика | Значение |
|---|---:|
| test loss | 0.5989 |
| accuracy | 0.8173 |
| precision | 0.8211 |
| recall | 0.7319 |
| F1 | 0.7739 |

## Итоговое сравнение

| Модель | Accuracy | Precision | Recall | F1 |
|---|---:|---:|---:|---:|
| `ViT-B-16 baseline` | 0.9845 | 0.9854 | 0.9783 | 0.9818 |
| `ResNet-18 weighted + scheduler` | 0.9814 | 0.9925 | 0.9638 | 0.9779 |
| `ResNet-18 strong augmentation` | 0.9814 | 1.0000 | 0.9565 | 0.9778 |
| `ResNet-18 baseline` | 0.9721 | 0.9708 | 0.9638 | 0.9673 |
| `Custom CNN baseline` | 0.9195 | 0.8636 | 0.9638 | 0.9110 |
| `Custom CNN improved` | 0.8173 | 0.8211 | 0.7319 | 0.7739 |

Лучшая модель в текущем запуске — `ViT-B-16 baseline`, F1 = 0.9818.

## Анализ ошибок

Для лучшей модели построена confusion matrix и выведены примеры ошибочных предсказаний. Это позволяет увидеть, какие изображения модель путает между классами. Такой анализ важен для прикладной задачи сортировки продукции: разные типы ошибок могут иметь разную стоимость.

## Выводы

В работе построен полный pipeline классификации изображений: загрузка датасета, анализ структуры, подготовка данных, обучение, сравнение моделей и анализ ошибок.

Основные выводы:

1. Transfer learning дает лучший результат на небольшом датасете изображений.
2. `ViT-B-16 baseline` показал максимальный F1 = 0.9818.
3. Улучшения ResNet-18 через class weights и scheduler немного повысили F1 относительно baseline.
4. Собственная CNN работает как учебный baseline, но уступает предобученным моделям.
5. Сильные аугментации и weighted loss не всегда улучшают модель, особенно если архитектура обучается почти с нуля.

Практический вывод: для задачи классификации сортов фисташек наиболее рационально использовать предобученную модель и дообучать ее под конкретные классы.
